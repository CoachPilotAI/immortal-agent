# Example: Session Notes (Memory Triage in Action)

This shows what a real session notes file looks like after a productive work session. Notice what was saved and — more importantly — what was intentionally dropped.

---

**Why:** Context keeps dying during heavy sessions. Need the brain to survive compression.

## What Was NOT Saved (intentionally dropped)

- The 15 file reads during architecture research
- 3 failed attempts at getting the delegation prompt right
- npm install for Playwright MCP server
- Reading 5 competitor docs during competitive analysis
- Multiple back-and-forth messages refining the README wording
- The actual code/content of the portfolio card (saved that it was created, not what it contains)

**These were all intermediate steps.** The decisions and outcomes were saved. The path to get there was not.
